## 🎭 它们是谁？（顶级职场分工大揭秘）

在正式开机之前，你必须理解这两位核心成员的“职场分工”：

* **OpenClaw (全能管家) 🤵**：它是你的“CEO”和“调度指挥中心”。它负责连接全球各大模型（Kimi, DeepSeek, Claude），管理你的工作区，并为你提供联网搜索、本地文件读写等全套“生存技能”。
* **Claude Code (顶级打工人) 🧑‍💻**：它是你的“卷王助理”和“首席分析师”。它驻扎在你的系统终端里，逻辑推演能力极强，专门对付硬核代码重构、复杂算法分析和长文档深度处理。
* **神级协作**：管家负责监控情报，打工人负责把信息转化为代码或方案。这套组合拳下来，你的效率将彻底进入“自动驾驶”模式！📈🔥

## 🏗️ 第一阶段：打好神级地基（Node.js + Git）

想让 AI 听话，地基不稳必塌！AI 工具跑不起来，99% 都是环境配置的锅。

1.  **管理员权限是“最高通行证”** 🛡️：不要直接双击打开 PowerShell！请务必右键点击 Windows 开始按钮，选择 **“终端（管理员）”**。
2.  **Node.js (核心引擎)** 🟢：它是 AI 工具运行的空气和水。请前往官网下载 **LTS（长期支持）版**，安装时 **务必勾选 "Add to PATH"**。
3.  **Git (助理的手脚)** 🐙：Windows 11 默认通常未安装。没有 Git，Claude 没法干活，管家也拉不到 GitHub 上的神仙技能包。
4.  **地基体检** ✅：在管理员窗口输入 `node -v` 和 `git -v`。看到版本号稳稳弹出，说明地基已打牢。

---

## 🏗️ 第二阶段：部署全能管家（OpenClaw 篇）

管家不需要追求虚无缥缈的实验新版，我们要的是稳如老狗的“养老稳定版”！🧘‍♂️

1.  **驱魔仪式（彻底清理干扰）** 🧹：如果你之前安装失败过，请立刻去 `C:\Users\你的用户名\` 下把 **`.openclaw`** 文件夹彻底删除。
2.  **精准安装稳定版** 📦：在管理员窗口执行：
    `npm i -g openclaw@2026.2.19`
3.  **上岗面谈 (Onboard)** 🧭：输入 `openclaw onboard` 开始配置：
    * **服务商 (Provider)**：根据你的 API Key 选择 `Moonshot AI (Kimi)` 或 `Anthropic`。
    * **API Key**：必须去官网后台点击 **蓝色图标一键复制**。🚨 **严禁用鼠标手动拉选**，防带入空格导致 401 报错！
    * **技能 (Skills)**：建议选 `Default`，Hooks 直接选择 `Skip for now`。

## 🧠 第三阶段：部署顶级打工人（Claude Code 篇）

这是让 Claude 在你的本地终端火力全开的秘密武器！🎮

1.  **入职报到** 🚀：在管理员版 PowerShell 中执行官方安装指令。
2.  **高级“黑科技”配置** 🇨🇳：
    * **安装 ccswitch**：一键自由切换模型的“遥控器”。
    * **接入国产心脏**：去 `deepsafe` 等平台领 Key 丢进 `ccswitch` 接入 DeepSeek。
3.  **VS Code 梦幻联动** 💻：安装官方 **Claude Code 插件**，设置中文界面并实现免登录秒开！

---

## 👿 第四阶段：OpenClaw 专属 401 报错“降魔录” ⚡

如果你在网页端看到 **HTTP 401: Invalid Authentication**，别哭，那是管家和打工人“没对上暗号”。

> **🚨 终极前提**：配置前请先 **彻底退出** 所有“魔法”上网软件！

* **死穴一：底层保险箱 (Vault) 锁死** 💣：OpenClaw 会把你填错的钥匙死死锁在系统的加密 Vault 里。改 JSON 没用，必须重新运行 `openclaw onboard` 填入 **正确** 的钥匙强制覆盖！
* **死穴二：域名访问受阻 (Kimi 专属坑)** 🌐：默认配置里的 `.ai` 结尾（如 `api.moonshot.ai`）在国内极不稳定。**破解法**：在 `openclaw.json` 里手动把 `moonshot.ai` 改成 **`.cn`**！
* **死穴三：Agent (代理层) 配置不匹配** 🤖：
    1.  **API 协议对齐**：Kimi 必须锁定为 `"api": "openai-completions"`。
    2.  **模型名对齐**：检查 JSON 底部 `agents` 里的 `primary` 模型名，必须和你网关层配置的名字一字不差！
    3.  **语法纠错**：遇到 `doctor --fix` 无限死循环时，手动去 JSON 删掉 `auth` 下多余的 `apiKey`。
